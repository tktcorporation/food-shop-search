# アプリケーション デザイン方針

このドキュメントは、グルメスポット検索アプリのデザイン方針・方向性をまとめたものです。

## サービスコンセプト

### ターゲットユーザー

**日常使い** - ランチ・ディナーをサクッと探したい人

- 忙しいビジネスパーソン
- 「今すぐ近くで食べたい」というニーズ
- 迷わず、素早く決めたい

### 目指す雰囲気

**クリーン × ポップ** の中間

- クリーン: 無駄がなく、情報が整理されている
- ポップ: 親しみやすく、使っていて楽しい
- 参考イメージ: LUUP（白ベース + 鮮やかなアクセント + ホワイトスペース）

## 視覚的優先順位

ユーザーが画面を見たとき、以下の順序で視線が誘導されるべき：

```
1位: 位置情報（現在地 / 選択された駅名）
     → 「どこのお店を見ているか」が一瞬でわかる

2位: 検索結果（お店のカード）
     → メインコンテンツ、選択肢

3位: フィルター・その他UI
     → 必要なときだけ使う、控えめに
```

## カラーパレット方針

### メインカラー: オレンジ系

- ポップでエネルギッシュな印象
- 食欲をそそる暖色系
- 飲食店検索サービスとしての親和性

### 配色ルール

| 用途             | 色                   | 意図                   |
| ---------------- | -------------------- | ---------------------- |
| 背景             | 白 (#FFFFFF)         | クリーンな印象のベース |
| メインアクション | 鮮やかオレンジ       | 位置情報、主要CTA      |
| テキスト         | ダークグレー         | 読みやすさ重視         |
| サブテキスト     | ミディアムグレー     | 補足情報               |
| カード背景       | 白 + 薄いボーダー    | 検索結果を際立たせる   |
| 成功/営業中      | グリーン             | ポジティブな状態       |
| エラー/警告      | レッド               | 注意喚起               |

### 視覚的階層

```
【強】位置情報バー ─ オレンジ背景 + 白文字（一番目立つ）
 ↓
【中】お店カード ─ 白背景 + 影 + 明確なボーダー
 ↓
【弱】フィルター等 ─ グレー系、控えめに
```

## 設計原則

### 1. メリハリを明確に

- 重要な要素は強く、そうでない要素は控えめに
- 同系色の濃淡だけでなく、明確なコントラストを使う

### 2. 視線誘導を意識

- ユーザーがどこから見ればいいかを迷わせない
- ファーストビューで「ここを見る」がわかる

### 3. シンプルに保つ

- 装飾より機能
- 余白を恐れない
- 要素を増やすより、既存要素を整理

## 避けるべきこと

- 同系色だけで構成（メリハリがなくなる）
- 過度な装飾（グラデーション、グロー効果）
- 情報の詰め込みすぎ
- 視覚的優先順位が不明確なレイアウト
